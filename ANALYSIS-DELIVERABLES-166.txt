KEY FINDINGS SUMMARY
================================================================================

ROOT CAUSE:
-----------
Circular dependency between health check system and connection filtering.

File: packages/agentc2/src/mcp/client.ts, lines 2818-2827
  → getIntegrationConnections() filters out connections with errorMessage
  
File: apps/agent/src/lib/inngest-functions.ts, line 8325
  → Health check preserves errorMessage when tools not found (because filtered)

IMPACT:
-------
• Severity: HIGH (affects all MCP integrations)
• Scope: Agent resolution, skill loading, provisioning, rediscovery, health checks
• Providers: HubSpot, Jira, Slack, GitHub, Firecrawl, JustCall, Playwright, ATLAS, custom
• User Experience: Tools unavailable despite UI showing "Connected"
• Recovery: None (until manual intervention or fix deployed)

THE FIX:
--------
Clear errorMessage before health checks (3 changes, 18 lines):
  1. Clear errors before calling listMcpToolDefinitions (5 lines added)
  2. Always clear errorMessage when no server error (1 word changed)
  3. Invalidate MCP cache after clearing (2 lines added)

Pattern matches manual test route (already proven in production).
