## 1. Purpose

This policy defines acceptable and prohibited uses of AgentC2 information systems, data, and resources by employees, contractors, and authorized users.

## 2. Scope

All employees, contractors, consultants, and third parties with access to Company systems, applications, data, networks, or devices.

## 3. Acceptable Use

### 3.1 General

- Use Company systems and data only for authorized business purposes.
- Protect credentials â€” do not share passwords, API keys, or access tokens.
- Lock workstations when unattended.
- Report suspected security incidents immediately to the CISO.
- Follow the Data Classification Policy when handling sensitive data.
- Use encrypted communication channels for sensitive information.

### 3.2 Email and Communication

- Use Company communication tools for business purposes.
- Do not send sensitive or classified data via unencrypted channels.
- Exercise caution with attachments and links from unknown sources.
- Do not use Company email for personal solicitation or marketing.

### 3.3 Software and Development

- Use only approved software and tools on Company systems.
- Follow secure development practices (code review, testing, no hardcoded secrets).
- Do not install unauthorized software on production or development systems.
- Report discovered vulnerabilities through the proper channel (CISO).

### 3.4 Data Handling

- Handle customer data only as required for your role.
- Do not copy customer data to personal devices or unauthorized storage.
- Follow data retention and disposal procedures.
- Do not access customer data without a legitimate business reason.
- Accessing customer accounts for support/debugging purposes must be logged.

## 4. Prohibited Use

The following activities are strictly prohibited:

### 4.1 Security Violations

- Attempting to access systems, data, or accounts without authorization.
- Circumventing or disabling security controls, access restrictions, or monitoring.
- Sharing credentials, tokens, or access keys with unauthorized parties.
- Installing malware, keyloggers, or surveillance software.
- Performing security testing without written authorization.

### 4.2 Data Violations

- Exfiltrating Company or customer data for unauthorized purposes.
- Storing credentials, API keys, or secrets in source code or unencrypted files.
- Accessing customer data without a legitimate business need.
- Disclosing confidential information to unauthorized parties.

### 4.3 AI System Misuse (Platform-Specific)

- Using AI agents to generate content that promotes violence, discrimination, or illegal activity.
- Impersonating individuals or organizations using AI agents without authorization.
- Conducting unauthorized surveillance or data collection via agent tools.
- Making automated decisions with legal or significant effects without human oversight.
- Circumventing or testing guardrail protections outside of authorized security testing.
- Using voice capabilities to create non-consensual deepfakes or impersonations.
- Sending unsolicited communications (spam) via connected integrations.
- Processing data of children under applicable age thresholds without appropriate safeguards.

### 4.4 Infrastructure Violations

- Using Company resources for cryptocurrency mining.
- Using Company resources for personal projects without approval.
- Intentionally degrading system performance or availability.
- Modifying production systems without following change management procedures.

## 5. Enforcement

- Violations will be investigated and may result in disciplinary action, up to and including termination.
- Violations involving illegal activity may be reported to law enforcement.
- Suspected violations should be reported to the CISO.

## 6. Acknowledgment

All employees and contractors must acknowledge this policy upon onboarding and annually thereafter.
