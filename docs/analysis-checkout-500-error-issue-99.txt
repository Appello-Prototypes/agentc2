--------------------------------------------------------------------------------
ROOT CAUSE (ONE SENTENCE)
--------------------------------------------------------------------------------

authenticateRequest() mixes synchronous cookie access (request.cookies.get())
with asynchronous cookie access (await cookies() inside getUserOrganizationId),
which Next.js 16 forbids, causing a runtime error that returns 500.

--------------------------------------------------------------------------------
THE FIX (THREE LINES)
--------------------------------------------------------------------------------

File: apps/agent/src/lib/api-auth.ts
Lines: 147-150

REMOVE:
    const preferredOrgId =
        request?.headers.get("x-organization-id")?.trim() ||
        request?.cookies.get("agentc2-active-org")?.value?.trim() ||
        null;

REPLACE WITH:
    const preferredOrgId = request?.headers.get("x-organization-id")?.trim() || null;

(Cookie is still read - asynchronously inside getUserOrganizationId)
