Implementation
Step-by-Step

Step 1: Core Workflow Cancel API Route

File: apps/agent/src/app/api/workflows/[slug]/runs/[runId]/cancel/route.tsPattern: apps/agent/src/app/api/agents/[id]/runs/[runId]/cancel/route.ts
Key Imports
import { NextRequest, NextResponse } from "next/server"
import { prisma, Prisma } from "@repo/database"
import { authenticateRequest } from "@/lib/api-auth"
import { refreshWorkflowMetrics } from "@/lib/metrics"
Logic
  1. Authenticate via authenticateRequest(request)
  2. Find workflow by slug, scoped to org: prisma.workflow.findFirst({ where: { OR: [{ slug }, { id: slug }], workspace: { organizationId } } })
  3. Find run by runId, verify it belongs to the workflow
  4. Guard: status must be QUEUED, RUNNING, or suspendedStep non-null
  5. Update WorkflowRun: status → CANCELLED, completedAt → now(), clear suspendedAt, suspendedStep, suspendDataJson → Prisma.DbNull
  6. Reject all pending ApprovalRequest records: updateMany({ where: { workflowRunId, status: "pending" }, data: { status: "rejected", decidedBy: "system", decidedAt: now(), decisionReason: "Workflow run cancelled by operator" } })
  7. Call refreshWorkflowMetrics(workflow.id, new Date())
  8. Return { success: true, cancelled: true, run: { id, status, completedAt } }
Code Condition
401 Unauthenticated
404 Workflow or run not found
409 Run not in cancellable state (already COMPLETED, FAILED, or CANCELLED)
500 Unexpected error

Step 2: Review Cancel Convenience Route

File: apps/agent/src/app/api/reviews/[id]/cancel/route.tsPattern: apps/agent/src/app/api/reviews/[id]/steps/route.ts
Logic
  1. Authenticate, find ApprovalRequest by id (scoped to org), including workflowRun.workflow
  2. Guard: workflowRunId must exist, run must be cancellable
  3. Perform same cancel mutations as Step 1 inline (not HTTP redirect)
  4. Return same response shape

Step 3: Add handleCancelRun to useReviews Hook

File: apps/agent/src/app/command/hooks/useReviews.ts
Interface
async function handleCancelRun(review: ReviewItem): Promise<void>
Logic
  1. POST ${getApiBase()}/api/reviews/${review.id}/cancel
  2. On success: addToast("Run cancelled -- {workflowName}"), remove review from list, clear selection, call fetchMetrics()
  3. On failure: addToast(data.error || "Cancel failed", "error")
  4. On network error: addToast("Network error", "error")

Step 4: Add Cancel UI to WorkflowStepsCard

File: apps/agent/src/app/command/components/WorkflowStepsCard.tsx
New Props
interface WorkflowStepsCardProps {
    reviewId: string
    onCancelRun?: () => void
    cancellingRun?: boolean
    runCancelled?: boolean
}
Logic
  1. Add confirmingCancel local state
  2. "Cancel Run" button in steps header -- only when onCancelRun provided and !runCancelled
  3. Click toggles confirmingCancel; inline confirmation bar appears below stepper
  4. "Keep Running" dismisses; "Yes, Cancel Run" calls onCancelRun()
  5. When cancellingRun: show spinner in button
  6. When runCancelled: show cancelled banner, suspended dot becomes cancelled style

Step 5: Wire Cancel in DecisionCard + Page

File: DecisionCard.tsx + page.tsx
Logic
  1. Add onCancelRun to DecisionCardProps
  2. Add local cancellingRun, runCancelled state in DecisionCard
  3. Pass to WorkflowStepsCard with async wrapper
  4. When runCancelled: hide action buttons
  5. In page.tsx: destructure handleCancelRun from useReviews(), pass to ReviewCardList and DecisionCard

Data Flow
Call Chain
User clicks "Cancel Run"
    └─ WorkflowStepsCard: setConfirmingCancel(true) → shows confirmation bar

User clicks "Yes, Cancel Run"
    └─ WorkflowStepsCard: calls onCancelRun()
        └─ DecisionCard: setCancellingRun(true), calls parent onCancelRun(review)
            └─ useReviews.handleCancelRun(review)
                └─ POST /api/reviews/{id}/cancel
                    ├─ Find ApprovalRequest → WorkflowRun → Workflow
                    ├─ WorkflowRun.status → CANCELLED
                    ├─ ApprovalRequest.status → rejected (updateMany)
                    └─ refreshWorkflowMetrics()
                └─ On success: addToast(), remove card from reviews list
            └─ DecisionCard: setRunCancelled(true), setCancellingRun(false)
                ├─ WorkflowStepsCard: shows cancelled banner
                ├─ DecisionCard: hides action buttons
                └─ Card fades from pending list
