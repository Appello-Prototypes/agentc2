Logic
Auto-Detection Rules
The StepDetailPanel component inspects each field in outputJson and renders it based on its type and name pattern. No backend changes required.
Detection Pattern Rendering
URLs Value matches ^https?:// Clickable link chips with auto-detected icon (GitHub, Cursor, etc.)
Issue / PR refs Key is issueUrl, prUrl, issueNumber, prNumber Link chip with issue/PR number (hidden if null)
Summary text Key ends with Summary or is text, response, result Styled summary block, first line bolded
Duration fields Key ends with DurationMs Human-readable time in metadata row
Agent IDs Key ends with AgentId Cursor agent session link chip
Branch names Key contains Branch Monospace text in metadata row
Decisions Key is decision, approved Decision chip (approved/rejected)
Errors error truthy or errorJson present Red error block with parsed validation errors
Null values Value is null Hidden entirely
Fallback Everything else Key-value audit grid + raw JSON toggle
Frontend-only change. No API or database modifications. The /api/reviews/[id]/steps route already returns outputJson -- the frontend renders it smarter using the StepItem component's native expandable + children pattern.
Architecture
Component Integration
Component / Function Purpose
StepItem (existing) UI component in @repo/ui. Already supports expandable prop + children. Renders chevron icon, handles expand/collapse state internally. No changes needed.
StepDetailPanel (new) New function component inside WorkflowStepsCard.tsx. Receives StepData, renders the structured detail view. Passed as children to each StepItem.
extractLinks(output) Scans for URL values, returns { label, url, icon }[]
extractMetadata(output) Pulls repository, branch, durations, channels into metadata items
extractSummaries(output) Finds *Summary / text / response fields, returns labeled text blocks
extractDecision(output) Detects decision/approved fields for human steps
extractError(output, errorJson) Parses error + validation fields into structured display
classifyUrl(url) Maps domains to icons (github.com, cursor.com, etc.)
