Implementation
Step-by-Step

Step 1: Add Extraction Helper Functions

File: apps/agent/src/app/command/components/WorkflowStepsCard.tsx
Interfaces
interface LinkItem { url: string; label: string; icon: string }
interface MetadataItem { label: string; value: string; mono?: boolean }
interface SummaryItem { label: string; text: string }
interface DecisionInfo { decision: string; approved: boolean; decidedBy?: string; channel?: string }
interface ErrorInfo { message: string; validationErrors?: { field: string; errors: string[] }[] }
Functions to Add (signatures)
function classifyUrl(url: string): { icon: string; label: string }
function extractLinks(output: unknown): LinkItem[]
function extractMetadata(output: unknown): MetadataItem[]
function extractSummaries(output: unknown): SummaryItem[]
function extractDecision(output: unknown): DecisionInfo | null
function extractError(output: unknown, errorJson: unknown): ErrorInfo | null
function getRemainingFields(output: unknown, handledKeys: Set<string>): Record<string, unknown>
Logic
  1. classifyUrl: map github.com → 🐙, cursor.com/agents → 🔧, default → 🔗 + hostname
  2. extractLinks: scan all fields for ^https?:// values; pair issueUrl+issueNumber, prUrl+prNumber; skip nulls; keys ending in AgentId become cursor links
  3. extractMetadata: pull repository, *Branch, *DurationMs (humanize with existing formatDuration), channel, decidedBy, issueNumber
  4. extractSummaries: keys ending in Summary or named text/response/result
  5. extractDecision: detect decision/approved fields
  6. extractError: if output.error truthy, parse message + validationErrors._errors; also use errorJson if present
  7. Each extractor adds consumed keys to a shared Set<string>; getRemainingFields returns unconsumed keys for the raw toggle

Step 2: Create StepDetailPanel Component

File: WorkflowStepsCard.tsx (inline component)
Interface
function StepDetailPanel({ step, suspendedStep }: {
    step: StepData
    suspendedStep?: string | null
})
Logic
  1. Call all extractors on step.outputJson, building handledKeys set
  2. Render metadata row if any metadata items
  3. Render links section if any links
  4. Render decision chip if extractDecision returns non-null
  5. Render error block if extractError returns non-null
  6. Render each summary in its own labeled section
  7. Always render raw JSON toggle at bottom with remaining field count

Detection
Auto-Detection Rules (Priority Order)
Extractors run in this order. Each adds consumed keys to a Set to prevent rendering the same field twice.
Priority Detection Pattern Rendering
1 Links Value matches ^https?:// or key ends in AgentId Clickable link chips with auto-detected icon
2 Metadata repository, *Branch, *DurationMs, channel, decidedBy, issueNumber, prNumber Key-value pairs in metadata row
3 Decision decision, approved Decision chip (green/red)
4 Error error truthy, message, validationErrors Red error block with field-level details
5 Summaries Key ends in Summary or is text/response/result Formatted text block, first line bolded
6 Null values Value is null Hidden entirely
7 Remaining Fallback Raw JSON toggle at bottom
Edge Cases
Behavioral Details
Scenario Behavior
outputJson is null Show "No output recorded" placeholder
outputJson is a plain string Render as summary text (not JSON)
outputJson is {} Show "Empty output" placeholder
All fields consumed by extractors Raw toggle shows "(all fields rendered above)"
URL value is null (e.g., prUrl: null) Skip -- don't render link chip
Step has both outputJson error and errorJson Show outputJson error first, then errorJson below
Suspended/pending steps Not expandable -- no chevron, no click handler
Multiple steps expanded simultaneously Allowed -- connector lines stretch for each
Very long summary (>2000 chars) Already truncated by API; render with max-height scroll
Architecture
Component Structure
Component / Function Purpose
StepItem (existing, @repo/ui) Supports expandable + children natively. Renders chevron, handles expand/collapse. No changes.
StepDetailPanel (new) Inline component in WorkflowStepsCard.tsx. Receives StepData, renders structured view. Passed as children to StepItem.
classifyUrl(url) Maps domains to icons: github.com, cursor.com, etc.
extractLinks(output) Scans for URL values, returns LinkItem[]
extractMetadata(output) Pulls repo, branch, duration, channel into metadata row
extractSummaries(output) Finds *Summary / text / response fields
extractDecision(output) Detects decision/approved for human steps
extractError(output, errorJson) Parses error + validation fields
getRemainingFields(output, handled) Returns unconsumed keys for raw JSON fallback
Data Flow
Call Chain
User clicks step row (e.g., "Output Summary")
    └─ StepItem (expandable=true): toggles internal expanded state
        └─ Renders children when expanded:
            └─ <StepDetailPanel step={step} suspendedStep={suspendedStep} />
                ├─ extractLinks(step.outputJson) → LinkItem[]
                ├─ extractMetadata(step.outputJson) → MetadataItem[]
                ├─ extractSummaries(step.outputJson) → SummaryItem[]
                ├─ extractDecision(step.outputJson) → DecisionInfo | null
                ├─ extractError(step.outputJson, step.errorJson) → ErrorInfo | null
                └─ getRemainingFields(step.outputJson, handledKeys) → raw JSON toggle

User clicks same step row again
    └─ StepItem: collapses, children unmounted
