Pipeline Swimlanes

A factory floor layout where each workflow type is a horizontal conveyor belt. Items flow left-to-right through stage columns, accumulating at human approval gates. The operator sees all workflows simultaneously and can batch-approve items at any gate with a single click.

ChipData 
interface ChipData {
  review: ReviewItem         // existing type
  steps: StepData[]          // fetched from /api/reviews/[id]/steps
  currentStage: string       // "triage" | "analysis" | "implementation" | "review-gate" | ...
  isAtGate: boolean          // true if currentStage includes "gate"
  riskLevel: string          // from reviewContext.riskLevel
  label: string              // "#202" or issue title truncated
  age: string                // "16h ago"
}

// Example chip:
{
  review: { id: "cmmo5sbm701q...", status: "pending", workflowSlug: "sdlc-bugfix-...", ... },
  steps: [{ stepId: "create-github-issue", status: "COMPLETED", ... }, ...],
  currentStage: "review-gate",
  isAtGate: true,
  riskLevel: "medium",
  label: "#202",
  age: "16h ago"
}
StageDefinition 
interface StageDefinition {
  id: string                 // "review-gate"
  label: string              // "Review Gate"
  isGate: boolean            // true
  stepPatterns: string[]     // ["merge-review", "audit-approval", "human-*"]
}

Step 1: usePipelineGrouping Hook

File: apps/agent/src/app/command/hooks/usePipelineGrouping.ts Pattern: apps/agent/src/app/command/hooks/useReviews.ts
Interface
function usePipelineGrouping(
  reviews: ReviewItem[],
  stepCache: Map<string, StepData[]>
): {
  pipelines: PipelineGroup[]
  gateCount: number
  stageColumns: StageDefinition[]
}
Logic
  1. Accept reviews + prefetched step data from existing useReviews hook
  2. Group reviews by workflowSlug into pipeline buckets
  3. For each review, find the last non-QUEUED step and map its stepId to a stage via StageDefinition.stepPatterns (glob matching)
  4. Classify items at gate stages as isAtGate: true
  5. Return sorted pipelines (most active first), total gate count, stage column definitions
  6. Memoize with useMemo keyed on review IDs and step cache size

Edge Cases
Boundary Conditions
Scenario Backend Frontend
0 active reviews Empty array returned Show "All pipelines are clear" empty state with gear icon
50+ items in one pipeline No change (paginated) Horizontal scroll within cells; chip count badge replaces individual chips after 8
Unknown workflow slug No change Group under "Other" pipeline row
Step data not yet loaded Returns empty steps array Show chip in "Triage" column (leftmost) with loading shimmer
Steps don't match any stage pattern No change Place chip in nearest stage by step index position
Concurrent approve while viewing board Review removed from pending Chip animates out of gate column; counter decrements
Workflow with no gate steps No change Pipeline row has no highlighted columns; items flow through without stopping

Data Flow
Call Chain
User opens Command page with "Pipeline" view toggle
    └─ page.tsx: viewMode === "pipeline"
        └─ useReviews().reviews → all pending items
            └─ useStepPrefetch(reviews) → batch GET /api/reviews/[id]/steps
                └─ Returns Map<reviewId, StepData[]>
        └─ usePipelineGrouping(reviews, stepCache)
            └─ Groups by workflowSlug → PipelineGroup[]
            └─ Maps last active step to stage column
        └─ <SwimlaneBoardView pipelines={...} />
            └─ CSS Grid renders rows x columns
            └─ Gate columns highlighted, batch bar at bottom

User clicks chip "#202"
    └─ PipelineChip.onClick(chipData)
        └─ SwimlaneBoardView: setSelectedChip(chipData)
            └─ ChipSlideout opens (Sheet component)
                └─ Renders context, links, summary, step timeline
                └─ Action buttons: Approve / Reject / Feedback

User clicks "Approve All at Review Gate"
    └─ SwimlaneBoardView: batchApprove(gateReviewIds)
        └─ useReviews().handleBatchApprove(ids)
            └─ POST /api/reviews/[id]/approve for each
            └─ Chips animate out; gate counter decrements
