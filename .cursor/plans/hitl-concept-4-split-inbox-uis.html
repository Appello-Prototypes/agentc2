Concept 4 of 5

Split Inbox

A master-detail layout inspired by Superhuman and Linear. The left panel is a compact, keyboard-navigable list of all items. The right panel shows the selected item's full workflow detail with step-by-step progress and action buttons. Designed for maximum throughput: an operator processing 50+ items/day can triage without ever touching the mouse.

Step 1: useKeyboardNav Hook

File: apps/agent/src/app/command/hooks/useKeyboardNav.ts
Interface
interface UseKeyboardNavOptions {
  items: ReviewItem[]
  selectedIndex: number
  onSelect: (index: number) => void
  onApprove: (reviewId: string) => void
  onReject: (reviewId: string) => void
  onFeedback: (reviewId: string) => void
  onCancel: (review: ReviewItem) => void
  enabled: boolean
}

function useKeyboardNav(options: UseKeyboardNavOptions): void
Logic
  1. Register keydown listener on window when enabled
  2. j = select next item (clamp to last), k = select previous (clamp to 0)
  3. a = approve current, r = reject current, f = feedback current, x = cancel current
  4. After action (approve/reject): auto-advance to next item in list
  5. Shift+A = batch approve all checked items
  6. Esc = clear selection/batch, ? = toggle help overlay
  7. Disable shortcuts when search input is focused

Logic
  1. Filter pills: All, Pending, Running, Done (replace tabs)
  2. Search input with Cmd+K focus shortcut
  3. Each row: checkbox, risk dot, ID, title (truncated), step badge, age
  4. Selected item highlighted with teal left border
  5. Age colored red if >2h for urgency visibility
  6. Status bar at bottom: item count + keyboard shortcut hints
  7. Scroll selected item into view when navigating with j/k

Data Flow
Call Chain
User opens Command with "Inbox" view
    └─ page.tsx: viewMode === "inbox"
        └─ useReviews() → all reviews
        └─ <SplitInboxView reviews={...} />
            └─ InboxList: renders filtered item list
            └─ InboxDetail: renders selected item detail + steps

User presses "j" to move down
    └─ useKeyboardNav: selectedIndex++
        └─ InboxList: highlights new item, scrolls into view
        └─ InboxDetail: fetches steps for new selection, renders detail

User presses "a" to approve
    └─ useKeyboardNav: onApprove(currentReviewId)
        └─ useReviews().handleApprove(id)
            └─ POST /api/reviews/[id]/approve
            └─ Item removed from list, selectedIndex auto-advances
            └─ InboxDetail refreshes to next item
