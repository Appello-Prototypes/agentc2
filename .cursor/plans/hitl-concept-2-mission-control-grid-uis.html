Mission Control Grid

A dense monitoring dashboard inspired by mission control rooms. Every active workflow run is a compact tile showing progress, current step, risk, and age at a glance. Primary actions (approve/reject) are inline on each tile -- zero clicks to take action. Click for deep detail in a side panel.

Architecture
Data Flow
No new API routes. Tiles are rendered from existing ReviewItem data. Step data is prefetched to compute progress bars. Tiles auto-sort by urgency (risk x age).

Step 1: useUrgencySort Hook

File: apps/agent/src/app/command/hooks/useUrgencySort.ts
Interface
interface UrgencySorted {
  attention: TileData[]  // waiting + failed, sorted by age desc
  running: TileData[]    // RUNNING status
  completed: TileData[]  // COMPLETED/CANCELLED today
}

interface TileData {
  review: ReviewItem
  steps: StepData[]
  progress: { completed: number; total: number; percent: number }
  urgencyScore: number  // riskWeight * ageMinutes
  currentStep: { name: string; status: string } | null
}
Logic
  1. Compute progress for each review from step data (completed / total)
  2. Calculate urgency score: riskWeight (high=3, medium=2, low=1) * ageMinutes
  3. Group into three bands: attention (status=pending/failed), running (RUNNING), completed (today)
  4. Sort attention band by urgency score descending (most urgent first)

Step 2: ControlTile Component

File: apps/agent/src/app/command/components/ControlTile.tsx
Props
interface ControlTileProps {
  tile: TileData
  onApprove: (reviewId: string) => void
  onReject: (reviewId: string) => void
  onFeedback: (reviewId: string) => void
  onRetryStep: (reviewId: string, stepId: string) => void
  onSkipStep: (reviewId: string, stepId: string) => void
  onClick: (tile: TileData) => void
}
Logic
  1. Left border color indicates urgency: red=urgent/failed, amber=waiting, blue=running, green=done
  2. Progress bar with percentage fill and animated shimmer for running items
  3. Inline action buttons for attention items (approve/reject/feedback/retry/skip)
  4. Running tiles have no action buttons (read-only monitoring)
  5. Age badge in top-right, colored red if >2h, muted if recent
  6. Click anywhere (except action buttons) opens TileSlideout

Step 3: MissionControlGrid Component

File: apps/agent/src/app/command/components/MissionControlGrid.tsx
Logic
  1. Toolbar with filter pills: All, Needs Attention, Running, Failed
  2. Three collapsible sections with urgency-band headers
  3. CSS Grid with auto-fill, minmax(260px, 1fr) for responsive tile layout
  4. Keyboard shortcuts: number keys (1-9) select tile, a/r/f for approve/reject/feedback
  5. Waiting tiles have subtle pulse animation on border
Edge Cases
Boundary Conditions
Scenario Backend Frontend
0 items Empty array Rocket icon with "All systems nominal" message
50+ tiles Paginated reviews Grid scrolls; sections collapse independently; virtualize if >100
Step data loading Steps API pending Tile shows shimmer progress bar and "Loading..." step text
Approve removes tile Review status changes Tile fades out, grid re-flows, section count decrements
All items completed No pending reviews Only "Completed Today" section visible; attention/running sections hidden
Narrow viewport (<768px) No change Grid collapses to 1 tile per row; actions stack vertically
