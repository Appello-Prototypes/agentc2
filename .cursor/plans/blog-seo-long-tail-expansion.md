### Problems with Current Coverage

1. **All keywords are platform-centric** — every post is about "AI agents" generically. No posts target people searching for solutions to specific business problems.
2. **No use-case content** — zero posts targeting "AI for [industry]" or "AI for [job function]" queries.
3. **No integration-specific content** — you have 10+ integrations (HubSpot, Slack, Jira, Gmail, etc.) but zero blog posts targeting "[integration] + AI agent" searches.
4. **No "how to" tutorial content** — the "tutorials" are conceptual, not step-by-step. People searching "how to build X" want actionable content.
5. **No competitor/alternative content** — only 1 comparison post. Alternative-seeking searches are high-intent.
6. **No problem-specific content** — zero posts targeting pain-point searches like "reduce AI hallucinations" or "AI agent keeps giving wrong answers."
7. **Thin content** — most posts are 300-500 words. Google favors 1500-2500 word posts for competitive terms.

## Content Guidelines for Long-Tail SEO

### Word Count Targets

| Post Type           | Target Words | Sections |
| ------------------- | ------------ | -------- |
| How-to / Tutorial   | 2000-2500    | 5-7      |
| Use-case            | 1500-2000    | 4-5      |
| Comparison          | 2000-3000    | 6-8      |
| Problem/Pain-point  | 1500-2000    | 4-5      |
| Technical deep-dive | 2000-2500    | 5-7      |
| Thought leadership  | 1500-2000    | 4-5      |

Current posts average ~300-500 words. **Every post needs to be 3-5x longer** to compete.

### On-Page SEO Checklist (Per Post)

- [ ] Primary keyword in title (H1), meta description, first paragraph, and at least one H2
- [ ] 3-5 secondary keywords used naturally throughout
- [ ] Internal links to 3+ docs pages (already doing this via `relatedDocs`)
- [ ] Internal links to 2+ other blog posts (NOT currently doing this — add `relatedPosts` field)
- [ ] Meta description under 160 characters with primary keyword
- [ ] URL slug contains primary keyword
- [ ] At least one list or table (Google loves structured content)
- [ ] FAQ section at the bottom (targets "People Also Ask" featured snippets)

### JSON-LD Additions

Add `FAQPage` structured data for posts with FAQ sections. This directly targets "People Also Ask" boxes in Google:

```typescript
function faqJsonLd(items: Array<{ question: string; answer: string }>) {
    return {
        "@context": "https://schema.org",
        "@type": "FAQPage",
        mainEntity: items.map((item) => ({
            "@type": "Question",
            name: item.question,
            acceptedAnswer: {
                "@type": "Answer",
                text: item.answer
            }
        }))
    };
}
```

Add `Article` structured data to every blog post (currently missing — only breadcrumb JSON-LD exists):

```typescript
function articleJsonLd(post: BlogPost) {
    return {
        "@context": "https://schema.org",
        "@type": "Article",
        headline: post.title,
        description: post.description,
        datePublished: post.publishedAt,
        dateModified: post.updatedAt,
        author: {
            "@type": "Organization",
            name: "AgentC2"
        },
        publisher: {
            "@type": "Organization",
            name: "AgentC2",
            url: "https://agentc2.ai"
        }
    };
}
```
