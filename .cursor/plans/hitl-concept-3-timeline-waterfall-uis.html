Timeline Waterfall

A time-based Gantt chart where each workflow run is a horizontal track. Steps are colored segments laid out along a time axis. A "now" line shows the current moment. The operator instantly sees which items have been waiting longest, which are stuck, and how fast workflows typically complete -- like watching race cars on parallel tracks.

Step 1: useTimelineLayout Hook

File: apps/agent/src/app/command/hooks/useTimelineLayout.ts
Interface
interface TimelineLayout {
  rows: TimelineRowData[]
  timeScale: { startMs: number; endMs: number; durationMs: number }
  timeMarks: { label: string; percent: number }[]
  nowPercent: number
}

interface TimelineRowData {
  review: ReviewItem
  segments: SegmentData[]
  markerPosition: number  // percent along track
  markerStatus: "waiting" | "running" | "failed"
}

interface SegmentData {
  stepId: string
  stepName: string
  status: string
  leftPercent: number
  widthPercent: number
  color: "green" | "blue" | "amber" | "red" | "cyan" | "muted"
  tooltipInfo: { title: string; detail: string }
}
Logic
  1. Determine time scale: min(earliest step startedAt) to now
  2. For each review's steps, compute left% = (step.startedAt - scaleStart) / scaleDuration * 100
  3. Compute width% = step.durationMs / scaleDuration * 100 (min 1% for visibility)
  4. Color mapping: COMPLETED=green, RUNNING=blue, FAILED=red, SUSPENDED/waiting=amber/cyan
  5. Generate time marks at even intervals (e.g., every hour for 6h scale, every 10m for 1h zoom)
  6. Support zoom levels: 1h, 3h, 6h, 12h, 24h with proportional rescaling

interface TimelineRowProps {
  row: TimelineRowData
  onApprove: (reviewId: string) => void
  onReject: (reviewId: string) => void
  onRetryStep: (reviewId: string, stepId: string) => void
  onSkipStep: (reviewId: string, stepId: string) => void
  onCancelRun: (review: ReviewItem) => void
  onClick: (row: TimelineRowData) => void
}
Logic
  1. Left label: issue ID, name, risk dot
  2. Track area: absolutely positioned segments with left% and width%
  3. Pulsing marker at current position (waiting=amber, running=blue, failed=red)
  4. Hover on segment: tooltip with step name, type, duration
  5. Right column: contextual action buttons based on status

Step 3: TimelineWaterfall Component

File: apps/agent/src/app/command/components/TimelineWaterfall.tsx
Logic
  1. Header: time axis marks with dashed vertical gridlines
  2. "Now" line: vertical violet line at current time position
  3. Zoom controls: buttons to switch time scale (1h, 3h, 6h, 12h, 24h)
  4. Sort options: by start time, by wait time, by risk level
  5. Click row to open detail slideout (reuse TileSlideout from Concept 2 or ChipSlideout from Concept 1)
  6. Auto-refresh: poll for step updates every 30s to animate running segments
Edge Cases
Boundary Conditions
Scenario Backend Frontend
0 active runs Empty array Checkered flag icon with "No races in progress"
Step with no timestamps startedAt is null Show as thin muted segment at position 0; tooltip says "Not started"
Steps that took <1s durationMs is tiny Enforce minimum segment width of 1% for visibility
Run started 24h+ ago Old timestamps Extend time scale or auto-zoom to fit; show "24h+" badge on row
Many runs (30+) Paginated Virtualized scroll for rows; sticky header stays visible
Zoom causes overlap No change Segments clamp to track bounds; overflow shows "..." indicator
