# Agents API

Complete reference for all agent-related API endpoints. All endpoints are prefixed with `/agent/api/agents` and require authentication via Bearer token or session cookie.

## Authentication

All endpoints require authentication. Use one of these methods:

- **Bearer token**: `Authorization: Bearer $TOKEN`
- **Session cookie**: Include session cookie from workspace login

<Callout type="info">
    API tokens can be generated in workspace settings. Session cookies are automatically set when
    logging in via the workspace UI.
</Callout>

## List Agents

<ApiEndpoint method="GET" path="/agent/api/agents" />

List all agents accessible to the authenticated user. Returns both system agents and user-created agents.

### Query Parameters

| Parameter | Type    | Default | Description                                                                       |
| --------- | ------- | ------- | --------------------------------------------------------------------------------- |
| `active`  | boolean | `true`  | Filter to active agents only                                                      |
| `system`  | boolean | `false` | Include system agents                                                             |
| `detail`  | string  | —       | `"capabilities"` for skill breakdown, `"discover"` for lightweight discovery view |
| `exclude` | string  | —       | Exclude agent with this slug (used with `detail=discover`)                        |
| `keyword` | string  | —       | Search keyword (used with `detail=discover`)                                      |

### Example Request

```bash
curl "https://agentc2.ai/agent/api/agents?active=true&detail=capabilities" \
  -H "Authorization: Bearer $TOKEN"
```

### Response `200 OK`

```json
{
    "success": true,
    "count": 5,
    "agents": [
        {
            "id": "cm_abc123",
            "slug": "support-agent",
            "name": "Support Agent",
            "description": "Handles customer inquiries",
            "modelProvider": "openai",
            "modelName": "gpt-4o",
            "isActive": true,
            "visibility": "PRIVATE",
            "tools": [
                {
                    "id": "at_123",
                    "toolId": "web-fetch",
                    "agentId": "cm_abc123"
                }
            ],
            "createdAt": "2026-02-18T12:00:00.000Z",
            "updatedAt": "2026-02-18T12:00:00.000Z"
        }
    ],
    "source": "database"
}
```

### Errors

| Status | Error Code       | Description                       |
| ------ | ---------------- | --------------------------------- |
| `401`  | `UNAUTHORIZED`   | Missing or invalid authentication |
| `500`  | `INTERNAL_ERROR` | Server error                      |

## Create Agent

<ApiEndpoint method="POST" path="/agent/api/agents" />

Create a new agent with specified configuration. The agent will be created in the authenticated user's workspace.

### Request Body

| Field                  | Type     | Required | Default        | Description                                |
| ---------------------- | -------- | -------- | -------------- | ------------------------------------------ |
| `name`                 | string   | ✅ Yes   | —              | Display name for the agent                 |
| `slug`                 | string   | ❌ No    | Auto-generated | URL-safe identifier (unique per workspace) |
| `description`          | string   | ❌ No    | `null`         | Human-readable description                 |
| `instructions`         | string   | ✅ Yes   | —              | System instructions for the agent          |
| `instructionsTemplate` | string   | ❌ No    | `null`         | Template with `{{variable}}` placeholders  |
| `modelProvider`        | string   | ✅ Yes   | —              | `"openai"` or `"anthropic"`                |
| `modelName`            | string   | ✅ Yes   | —              | Model identifier (e.g., `"gpt-4o"`)        |
| `temperature`          | number   | ❌ No    | `0.7`          | Sampling temperature (0-2)                 |
| `maxTokens`            | number   | ❌ No    | `null`         | Maximum tokens in response                 |
| `modelConfig`          | object   | ❌ No    | `null`         | Provider-specific options                  |
| `memoryEnabled`        | boolean  | ❌ No    | `false`        | Enable conversation memory                 |
| `memoryConfig`         | object   | ❌ No    | `null`         | Memory configuration                       |
| `maxSteps`             | number   | ❌ No    | `5`            | Maximum tool call rounds per turn          |
| `tools`                | string[] | ❌ No    | `[]`           | Array of tool IDs to attach                |
| `subAgents`            | string[] | ❌ No    | `[]`           | Agent slugs this agent can delegate to     |
| `workflows`            | string[] | ❌ No    | `[]`           | Workflow IDs this agent can invoke         |
| `scorers`              | string[] | ❌ No    | `[]`           | Evaluation scorer names                    |
| `visibility`           | string   | ❌ No    | `"PRIVATE"`    | `"PRIVATE"` or `"PUBLIC"`                  |
| `metadata`             | object   | ❌ No    | `null`         | Custom JSON metadata                       |
| `isActive`             | boolean  | ❌ No    | `true`         | Whether the agent is active                |

### Example Request

```bash
curl -X POST https://agentc2.ai/agent/api/agents \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Research Assistant",
    "slug": "research-assistant",
    "instructions": "You are a research assistant. Search the web and provide well-sourced answers.",
    "modelProvider": "openai",
    "modelName": "gpt-4o",
    "temperature": 0.3,
    "memoryEnabled": true,
    "tools": ["web-fetch"]
  }'
```

### Response `200 OK`

```json
{
    "success": true,
    "agent": {
        "id": "cm_abc123",
        "slug": "research-assistant",
        "name": "Research Assistant",
        "instructions": "You are a research assistant...",
        "modelProvider": "openai",
        "modelName": "gpt-4o",
        "temperature": 0.3,
        "memoryEnabled": true,
        "tools": [
            {
                "id": "at_123",
                "toolId": "web-fetch",
                "agentId": "cm_abc123"
            }
        ],
        "isActive": true,
        "createdAt": "2026-02-18T12:00:00.000Z",
        "updatedAt": "2026-02-18T12:00:00.000Z"
    }
}
```

### Errors

| Status | Error Code                | Description                        |
| ------ | ------------------------- | ---------------------------------- |
| `400`  | `MISSING_REQUIRED_FIELDS` | Missing required fields in request |
| `400`  | `INVALID_MODEL`           | Model not found for provider       |
| `400`  | `INVALID_SLUG`            | Slug contains invalid characters   |
| `401`  | `UNAUTHORIZED`            | Missing or invalid authentication  |
| `409`  | `SLUG_CONFLICT`           | Slug already exists                |

## Update Agent

<ApiEndpoint method="PATCH" path="/agent/api/agents/{slug}" />

Update an existing agent. Only USER-type agents can be updated; SYSTEM agents are protected. All fields are optional—only include fields you want to update.

### Path Parameters

| Parameter | Type   | Description      |
| --------- | ------ | ---------------- |
| `slug`    | string | Agent slug or ID |

### Request Body

All fields are optional. Only include fields you want to update:

| Field           | Type     | Required | Description                 |
| --------------- | -------- | -------- | --------------------------- |
| `name`          | string   | ❌ No    | Display name                |
| `instructions`  | string   | ❌ No    | System instructions         |
| `temperature`   | number   | ❌ No    | Sampling temperature (0-2)  |
| `maxSteps`      | number   | ❌ No    | Maximum tool call rounds    |
| `tools`         | string[] | ❌ No    | Array of tool IDs           |

```bash
curl -X PATCH https://agentc2.ai/agent/api/agents/research-assistant \
  -H "Authorization: Bearer $TOKEN" \

  -d '{
    "temperature": 0.5,
```

### Response `200 OK`

```json
{
    "success": true,

        "id": "cm_abc123",
        "slug": "research-assistant",
```

## Chat with Agent

<ApiEndpoint method="POST" path="/agent/api/agents/{slug}/chat" />

Send a chat message to an agent and receive a streaming response. The agent will process the message using its configured tools and memory.

### Path Parameters

| Parameter | Type   | Description      |
| --------- | ------ | ---------------- |
| `slug`    | string | Agent slug or ID |

### Request Body

| Field            | Type   | Required | Description                                                 |
| ---------------- | ------ | -------- | ----------------------------------------------------------- |
| `messages`       | array  | ✅ Yes   | Array of message objects `[{role: "user", content: "..."}]` |
| `threadId`       | string | ❌ No    | Conversation thread ID (for memory)                         |
| `requestContext` | object | ❌ No    | Additional context (userId, metadata, etc.)                 |
| `modelOverride`  | object | ❌ No    | Override agent's model settings for this request            |

### Example Request

```bash
curl -X POST https://agentc2.ai/agent/api/agents/research-assistant/chat \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      { "role": "user", "content": "What are the latest AI developments?" }
    ],
    "threadId": "thread_123"
  }'
```

### Response `200 OK`

The response is a streaming Server-Sent Events (SSE) stream:

```
data: {"type":"text","delta":"The","text":"The"}

data: {"type":"text","delta":" latest","text":"The latest"}

data: {"type":"tool_call","toolName":"web-fetch","args":{"url":"https://..."}}

data: {"type":"tool_result","toolName":"web-fetch","result":"..."}

data: {"type":"text","delta":" developments","text":"The latest developments"}

data: {"type":"done","runId":"run_abc123"}
```

### Response Format

| Field      | Type   | Description                                        |
| ---------- | ------ | -------------------------------------------------- |
| `type`     | string | `"text"`, `"tool_call"`, `"tool_result"`, `"done"` |
| `delta`    | string | Incremental text (for `type: "text"`)              |
| `text`     | string | Full text so far (for `type: "text"`)              |
| `toolName` | string | Tool name (for `type: "tool_call"`)                |
| `args`     | object | Tool arguments (for `type: "tool_call"`)           |
| `result`   | any    | Tool result (for `type: "tool_result"`)            |
| `runId`    | string | Agent run ID (for `type: "done"`)                  |

### Errors

| Status | Error Code            | Description                       |
| ------ | --------------------- | --------------------------------- |
| `400`  | `INVALID_MESSAGES`    | Invalid messages format           |
| `401`  | `UNAUTHORIZED`        | Missing or invalid authentication |
| `404`  | `NOT_FOUND`           | Agent not found                   |
| `429`  | `RATE_LIMIT_EXCEEDED` | Rate limit exceeded               |
| `500`  | `INTERNAL_ERROR`      | Server error                      |

## Error Codes

All endpoints may return these error codes:

| Status | Error Code                | Description                        | Solution                          |
| ------ | ------------------------- | ---------------------------------- | --------------------------------- |
| `400`  | `MISSING_REQUIRED_FIELDS` | Missing required fields in request | Include all required fields       |
| `400`  | `INVALID_MODEL`           | Model not found for provider       | Check available models            |
| `400`  | `INVALID_SLUG`            | Slug contains invalid characters   | Use alphanumeric and hyphens only |
| `400`  | `INVALID_TOOL`            | Tool not found                     | Check tool ID is correct          |
| `400`  | `INVALID_MESSAGES`        | Invalid messages format            | Use correct message array format  |
| `401`  | `UNAUTHORIZED`            | Missing or invalid authentication  | Include `Authorization` header    |
| `403`  | `FORBIDDEN`               | Insufficient permissions           | Ensure you have access            |
| `404`  | `NOT_FOUND`               | Agent or resource not found        | Check slug/ID is correct          |
| `409`  | `SLUG_CONFLICT`           | Slug already exists                | Use a different slug              |
| `409`  | `TOOL_ALREADY_ATTACHED`   | Tool already attached              | Tool is already on agent          |
| `429`  | `RATE_LIMIT_EXCEEDED`     | Rate limit exceeded                | Wait before retrying              |
| `500`  | `INTERNAL_ERROR`          | Server error                       | Retry request or contact support  |

### Error Response Format

```json
{
    "success": false,
    "error": "Agent with slug 'support-agent' already exists",
    "code": "SLUG_CONFLICT"
}
```

## Rate Limits

API endpoints are rate-limited to prevent abuse:

- **Chat endpoints**: 60 requests per minute per IP
- **CRUD endpoints**: 100 requests per minute per user
- **List endpoints**: 200 requests per minute per user

Rate limit headers are included in responses:

```
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 45
X-RateLimit-Reset: 1642608000
```
