# MCP Tool Reference

The AgentC2 MCP server exposes 202 tools across seven categories. This page provides a categorized listing with tool names and descriptions.

<Callout type="tip">
    The canonical tool list with full parameter schemas is always available by connecting an MCP
    client and running `tools/list`. The schemas include input types, required fields, and
    descriptions for every parameter.
</Callout>

## Agent Management

Tools for creating, configuring, invoking, evaluating, and managing agents. This is the largest category with approximately 50 tools.

### Core agent operations

| Tool                   | Description                                                    |
| ---------------------- | -------------------------------------------------------------- |
| `agent_create`         | Create a new agent with instructions, model, and configuration |
| `agent_delete`         | Delete an agent and its associated data                        |
| `agent_discover`       | List all agents in the workspace with status and metadata      |
| `agent_invoke_dynamic` | Invoke any agent with a prompt and receive a response          |

### Budget and cost tracking

| Tool                  | Description                                                 |
| --------------------- | ----------------------------------------------------------- |
| `agent_budget_get`    | Get the current budget configuration for an agent           |
| `agent_budget_update` | Set or update monthly/daily budget limits                   |
| `agent_costs`         | Retrieve cost breakdown by agent, time period, or model     |
| `agent_analytics`     | View usage analytics including token counts and run volumes |

### Guardrails and governance

| Tool                      | Description                                         |
| ------------------------- | --------------------------------------------------- |
| `agent_guardrails_get`    | Get guardrail policies applied to an agent          |
| `agent_guardrails_update` | Update input/output guardrail rules                 |
| `agent_guardrails_events` | View guardrail trigger history and blocked requests |

### Evaluations and feedback

| Tool                     | Description                                  |
| ------------------------ | -------------------------------------------- |
| `agent_evaluations_list` | List evaluation runs for an agent            |
| `agent_evaluations_run`  | Execute an evaluation suite against an agent |
| `agent_feedback_submit`  | Submit human feedback on an agent response   |
| `agent_feedback_list`    | List feedback entries for an agent           |

### Learning and experiments

| Tool                         | Description                                        |
| ---------------------------- | -------------------------------------------------- |
| `agent_learning_experiments` | View and manage learning experiments and A/B tests |

### Skill attachment

| Tool                 | Description                  |
| -------------------- | ---------------------------- |
| `agent_attach_skill` | Attach a skill to an agent   |
| `agent_detach_skill` | Remove a skill from an agent |

### Individual agent tools

Each agent in your workspace also exposes a dedicated invocation tool (e.g., `agent_general_assistant`, `agent_research_agent`). These tools invoke a specific agent by slug, providing a shortcut compared to `agent_invoke_dynamic`.

| Tool                     | Description                                                 |
| ------------------------ | ----------------------------------------------------------- |
| `workflow_create`        | Create a new workflow definition                            |
| `workflow_read`          | Get a workflow's configuration and step definitions         |
| `workflow_update`        | Update a workflow's steps, configuration, or metadata       |
| `workflow_delete`        | Delete a workflow                                           |
| `workflow_generate`      | AI-generate a workflow from a natural language description  |
| `workflow_validate`      | Validate a workflow definition for structural correctness   |

| `workflow_get_run`       | Get the status and output of a specific workflow run        |

| `workflow_versions`      | List version history for a workflow                         |

## Discovering tools programmatically

Connect any MCP client and the full tool catalog is available via the `tools/list` MCP protocol method. This returns the complete list with:

- Tool names and descriptions
- Input parameter JSON schemas (types, required fields, defaults)
- Output schema descriptions

```bash
# Using Claude Code as an example
claude> "What tools do you have from the AgentC2 server?"
```
