# Channels Overview

Channels enable you to deploy agent logic across messaging platforms (Slack, WhatsApp, Telegram, voice, web embeds) with unified conversation handling, memory persistence, and multi-agent routing. Agents respond to users on their preferred platform while maintaining context across all channels.

## What are channels?

Channels are platform-specific integrations that connect AgentC2 agents to external messaging systems. They handle inbound events (messages, mentions, voice calls), normalize them into a unified format, route to the appropriate agent, format responses for the platform, and send replies back to users.

Channels abstract away platform-specific APIs, webhooks, and authentication, allowing you to deploy the same agent logic across multiple platforms without code changes.

## Channel architecture

Every channel follows a consistent architecture:

```
Inbound Event → Normalize → Agent Processing → Format Response → Send
     ↓              ↓              ↓                  ↓            ↓
Platform API    Unified      Agent Chat API    Platform-      Platform
Webhook         Format        (with memory)     specific       API
```

### 1. Inbound event

Platforms send events to AgentC2 webhooks:

- **Slack**: `POST /agent/api/slack/events` — Mentions, DMs, reactions
- **WhatsApp**: `POST /agent/api/whatsapp/webhook` — Messages, media
- **Telegram**: `POST /agent/api/telegram/webhook` — Messages, commands
- **Voice**: `POST /agent/api/voice/webhook` — Call events, audio streams
- **Web Embed**: `POST /agent/api/embed/chat` — Chat messages from embed widget

### 2. Normalize

Events are normalized into a unified format:

```json
{
    "channelType": "slack",
    "channelId": "C123456",
    "userId": "U789012",
    "message": "What is quantum computing?",
    "threadId": "thread_abc123",
    "metadata": {
        "platform": "slack",
        "teamId": "T123456",
        "channelName": "#general"
    }
}
```

### 3. Agent processing

Normalized events are routed to agents via the chat API:

```bash
POST /agent/api/agents/{slug}/chat
{
  "messages": [
    {"role": "user", "content": "What is quantum computing?"}
  ],
  "threadId": "thread_abc123",
  "requestContext": {
    "channelType": "slack",
    "userId": "U789012"
  }
}
```

Agents process requests with full access to:

- **Memory** — Conversation history and semantic recall
- **Tools** — MCP servers, built-ins, external APIs
- **Multi-agent routing** — Delegate to specialized agents

### 4. Format response

Agent responses are formatted for the platform:

- **Slack**: Block Kit formatting, threads, reactions
- **WhatsApp**: Text, media, templates
- **Telegram**: Markdown, inline keyboards, media
- **Voice**: Text-to-speech, audio streaming
- **Web Embed**: HTML, markdown, streaming

### 5. Send

Formatted responses are sent via platform APIs:

- **Slack**: `chat.postMessage` API
- **WhatsApp**: Business API
- **Telegram**: Bot API
- **Voice**: Audio streaming
- **Web Embed**: WebSocket/SSE streaming

## Supported channels

AgentC2 supports the following channels:

| Channel       | Status        | Features                                                   | Setup Complexity |
| ------------- | ------------- | ---------------------------------------------------------- | ---------------- |
| **Slack**     | ✅ Production | Mentions, DMs, threads, reactions, multi-agent routing     | Medium           |
| **WhatsApp**  | ✅ Production | Messages, media, templates, delivery receipts              | High             |
| **Telegram**  | ✅ Production | Messages, commands, inline keyboards, media                | Low              |
| **Voice**     | ✅ Production | ElevenLabs, OpenAI Realtime, tool execution, interruptions | High             |
| **Web Embed** | ✅ Production | Chat widget, iframe/script tag, token auth, rate limiting  | Low              |

<Callout type="info">
    All channels support conversation memory, multi-agent routing (via `agent:` prefix), and tool
    execution. Each channel maintains separate conversation threads per user/platform combination.
</Callout>

## Channel configuration

Channels are configured via environment variables and database settings:

### Environment variables

```bash
# Slack
SLACK_BOT_TOKEN="xoxb-..."
SLACK_SIGNING_SECRET="..."
SLACK_CLIENT_ID="..."
SLACK_CLIENT_SECRET="..."

# WhatsApp
WHATSAPP_API_KEY="..."
WHATSAPP_PHONE_NUMBER_ID="..."

# Telegram
TELEGRAM_BOT_TOKEN="..."

# Voice (ElevenLabs)
ELEVENLABS_API_KEY="..."
ELEVENLABS_AGENT_ID="..."
ELEVENLABS_WEBHOOK_SECRET="..."

# Web Embed
# No special env vars required (uses API tokens)
```

### Database configuration

Channel bindings are stored in `InstanceChannelBinding`:

```prisma
model InstanceChannelBinding {
    id                   String
    instanceId           String // Agent instance ID
    channelType          String // "slack", "whatsapp", "telegram", "voice", "web"
    channelIdentifier    String // Channel ID, phone number, token, etc.
    channelName          String? // Display name
    channelMetadata      Json? // Platform-specific config
    replyMode            String? // "thread", "channel", "ephemeral"
    responseLength       String? // "concise", "standard", "detailed"
    richFormatting       Boolean? // Platform-specific formatting
    triggerOnAllMessages Boolean // Respond to every message
    triggerKeywords      String[] // Keywords that trigger response
    isActive             Boolean
}
```

## Conversation memory

All channels maintain conversation memory:

- **Thread-based**: Each platform thread maintains its own conversation context
- **Cross-platform**: Same user across platforms can share memory (if configured)
- **Semantic recall**: Agents can recall relevant past conversations
- **Message history**: Last N messages included in context (configurable per channel)

### Memory configuration

```bash
# Per-channel memory settings
{
  "slack": {
    "lastMessages": 20,
    "semanticRecall": {"topK": 3}
  },
  "whatsapp": {
    "lastMessages": 10,
    "semanticRecall": {"topK": 2}
  }
}
```

## Multi-agent routing

All channels support routing to different agents using the `agent:` prefix:

- **Slack**: `@Bot agent:research What is quantum computing?`
- **WhatsApp**: `agent:research What is quantum computing?`
- **Telegram**: `/agent research What is quantum computing?`
- **Voice**: "Agent research, what is quantum computing?"
- **Web Embed**: Prefix message with `agent:research`

### Default agent

Each channel can have a default agent:

```bash
SLACK_DEFAULT_AGENT_SLUG="assistant"
WHATSAPP_DEFAULT_AGENT_SLUG="support"
TELEGRAM_DEFAULT_AGENT_SLUG="assistant"
```

If no `agent:` prefix is provided, the default agent handles the request.

## Channel lifecycle

Every channel follows a predictable lifecycle:

### 1. Setup

Configure platform credentials and webhooks:

```bash
# Slack: Create app, configure permissions, set webhook URL
# WhatsApp: Register business, configure webhook
# Telegram: Create bot, set webhook
# Voice: Configure ElevenLabs agent, set webhook
# Web Embed: Generate embed token
```

### 2. Bind agent

Create channel binding to connect agent to platform:

```bash
curl -X POST https://agentc2.ai/agent/api/instances/{instanceId}/bindings \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "channelType": "slack",
    "channelIdentifier": "C123456",
    "channelName": "#general",
    "replyMode": "thread"
  }'
```

### 3. Test

Send test messages and verify agent responses:

```bash
# Slack: @mention bot in channel
# WhatsApp: Send message to configured number
# Telegram: Send /start command
# Voice: Make test call
# Web Embed: Open embed widget and chat
```

### 4. Monitor

Track channel performance:

```bash
# List channel bindings
curl https://agentc2.ai/agent/api/instances/{instanceId}/bindings \
  -H "Authorization: Bearer $TOKEN"

# View channel metrics
curl https://agentc2.ai/agent/api/channels/{channelType}/metrics \
  -H "Authorization: Bearer $TOKEN"
```

## Channel capabilities

| Capability              | Description                                        | Supported Platforms       |
| ----------------------- | -------------------------------------------------- | ------------------------- |
| **Conversation Memory** | Maintain context across messages                   | All                       |
| **Multi-agent Routing** | Route to different agents via prefix               | All                       |
| **Tool Execution**      | Agents can use tools (MCP, built-ins)              | All                       |
| **Rich Formatting**     | Platform-specific formatting (Block Kit, markdown) | Slack, Telegram, Web      |
| **Media Support**       | Images, files, audio, video                        | WhatsApp, Telegram, Voice |
| **Threading**           | Reply in threads, maintain thread context          | Slack, Telegram           |
| **Reactions**           | Emoji reactions for feedback                       | Slack, Telegram           |
| **Streaming**           | Real-time response streaming                       | Web Embed, Voice          |
| **Rate Limiting**       | Prevent abuse with rate limits                     | All                       |
| **Authentication**      | Token-based or OAuth authentication                | All                       |
