# Skills API Reference

Complete reference for all skill-related API endpoints. All endpoints are prefixed with `/agent/api/skills` and require authentication via Bearer token or session cookie.

## Authentication

All endpoints require authentication. Use one of these methods:

- **Bearer token**: `Authorization: Bearer $TOKEN`
- **Session cookie**: Include session cookie from workspace login

<Callout type="info">
    API tokens can be generated in workspace settings. Session cookies are automatically set when
    logging in via the workspace UI.
</Callout>

## List Skills

<ApiEndpoint method="GET" path="/agent/api/skills" />

List all skills accessible to the authenticated user.

### Query Parameters

| Parameter     | Type   | Default       | Description                                         |
| ------------- | ------ | ------------- | --------------------------------------------------- |
| `category`    | string | —             | Filter by category (e.g., "communication", "crm")   |
| `type`        | string | —             | Filter by type (`"USER"` or `"SYSTEM"`)             |
| `tags`        | string | —             | Comma-separated tags to filter by                   |
| `skip`        | number | `0`           | Number of skills to skip (pagination)               |
| `take`        | number | `50`          | Maximum number of skills to return                  |
| `workspaceId` | string | Auto-detected | Workspace ID (defaults to user's default workspace) |

### Example Request

```bash
curl "https://agentc2.ai/agent/api/skills?category=communication&type=USER&take=20" \
  -H "Authorization: Bearer $TOKEN"
```

### Response

```json
{
    "success": true,
    "skills": [
        {
            "id": "cm123abc456",
            "slug": "email-drafting",
            "name": "Email Drafting",
            "description": "Draft professional emails",
            "category": "communication",
            "tags": ["email", "drafting"],
            "version": 1,
            "type": "USER",
            "createdAt": "2026-02-18T12:00:00.000Z"
        }
    ],
    "total": 15,
    "skip": 0,
    "take": 20
}
```

## Create Skill

<ApiEndpoint method="POST" path="/agent/api/skills" />

Create a new skill.

### Request Body

| Field          | Type     | Required | Default       | Description                                             |
| -------------- | -------- | -------- | ------------- | ------------------------------------------------------- |
| `slug`         | string   | ✅ Yes   | —             | URL-safe identifier (unique per workspace)              |
| `name`         | string   | ✅ Yes   | —             | Display name for the skill                              |
| `description`  | string   | ❌ No    | `null`        | Human-readable description                              |
| `instructions` | string   | ✅ Yes   | —             | Procedural knowledge describing how to perform the task |
| `examples`     | string   | ❌ No    | `null`        | Reference outputs or patterns                           |
| `category`     | string   | ❌ No    | `null`        | Category for organization                               |
| `tags`         | string[] | ❌ No    | `[]`          | Array of tags for filtering                             |
| `metadata`     | object   | ❌ No    | `null`        | Custom JSON metadata                                    |
| `type`         | string   | ❌ No    | `"USER"`      | `"USER"` or `"SYSTEM"`                                  |
| `workspaceId`  | string   | ❌ No    | Auto-detected | Workspace ID                                            |

### Example Request

```bash
curl -X POST https://agentc2.ai/agent/api/skills \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "slug": "email-drafting",
    "name": "Email Drafting",
    "description": "Draft professional emails",
    "instructions": "When drafting emails:\n1. Use clear subject lines\n2. Structure with paragraphs\n3. Include call-to-action",
    "category": "communication",
    "tags": ["email", "drafting"]
  }'
```

### Response

```json
{
    "success": true,
    "skill": {
        "id": "cm123abc456",
        "slug": "email-drafting",
        "name": "Email Drafting",
        "description": "Draft professional emails",
        "instructions": "When drafting emails:\n1. Use clear subject lines...",
        "category": "communication",
        "tags": ["email", "drafting"],
        "version": 1,
        "type": "USER",
        "workspaceId": "ws_abc123",
        "createdAt": "2026-02-18T12:00:00.000Z",
        "updatedAt": "2026-02-18T12:00:00.000Z"
    }
}
```

## Update Skill

<ApiEndpoint method="PUT" path="/agent/api/skills/{skillId}" />

Update an existing skill. Only USER-type skills can be updated; SYSTEM skills are protected.

### Path Parameters

| Parameter | Type   | Description      |
| --------- | ------ | ---------------- |
| `skillId` | string | Skill ID or slug |

### Request Body

All fields are optional. Only include fields you want to update:

| Field           | Type     | Description                             |
| --------------- | -------- | --------------------------------------- |
| `name`          | string   | Display name                            |
| `description`   | string   | Human-readable description              |
| `instructions`  | string   | Procedural knowledge                    |
| `examples`      | string   | Reference outputs                       |
| `category`      | string   | Category                                |
| `tags`          | string[] | Array of tags                           |
| `metadata`      | object   | Custom JSON metadata                    |
| `changeSummary` | string   | Description of changes (for versioning) |

### Example Request

```bash
curl -X PUT https://agentc2.ai/agent/api/skills/email-drafting \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "instructions": "Updated instructions with improved formatting guidelines",
    "changeSummary": "Improved formatting guidelines based on user feedback"
  }'
```

### Response

```json
{
    "success": true,
    "skill": {
        "id": "cm123abc456",
        "slug": "email-drafting",
        "instructions": "Updated instructions with improved formatting guidelines",
        "version": 2,
        "updatedAt": "2026-02-18T12:05:00.000Z"
    }
}
```

## Error Codes

All endpoints may return these error codes:

| Status | Error Code                | Description                        | Solution                          |
| ------ | ------------------------- | ---------------------------------- | --------------------------------- |
| `400`  | `MISSING_REQUIRED_FIELDS` | Missing required fields in request | Include all required fields       |
| `400`  | `INVALID_SLUG`            | Slug contains invalid characters   | Use alphanumeric and hyphens only |
| `401`  | `UNAUTHORIZED`            | Missing or invalid authentication  | Include `Authorization` header    |
| `403`  | `FORBIDDEN`               | Insufficient permissions           | Ensure you have access            |
| `404`  | `NOT_FOUND`               | Skill or resource not found        | Check ID/slug is correct          |
| `409`  | `SLUG_CONFLICT`           | Slug already exists                | Use a different slug              |
| `429`  | `RATE_LIMIT_EXCEEDED`     | Rate limit exceeded                | Wait before retrying              |
| `500`  | `INTERNAL_ERROR`          | Server error                       | Retry request or contact support  |

### Error Response Format

```json
{
    "success": false,
    "error": "Skill with slug 'email-drafting' already exists",
    "code": "SLUG_CONFLICT"
}
```

## Rate Limits

API endpoints are rate-limited to prevent abuse:

- **CRUD endpoints**: 100 requests per minute per user
- **List endpoints**: 200 requests per minute per user
- **Attachment endpoints**: 50 requests per minute per user

Rate limit headers are included in responses:

```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 45
X-RateLimit-Reset: 1642608000
```
