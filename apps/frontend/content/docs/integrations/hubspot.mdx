# HubSpot

HubSpot integration connects your agents to HubSpot CRM, enabling them to search contacts, manage companies, create deals, and track pipeline stages. The integration uses HubSpot's hosted MCP server, providing standardized tool access to CRM data.

## Overview

The HubSpot integration exposes CRM operations as MCP tools that agents can call during conversations. Agents can look up customer information, create deals, update contact properties, and search the pipeline—all through natural language requests.

<Callout type="info">
    HubSpot uses a hosted MCP server at `https://mcp.hubspot.com`. No self-hosting required—just
    provide your private app access token.
</Callout>

## Prerequisites

- HubSpot account with admin access
- Private app created in HubSpot (Settings > Integrations > Private Apps)
- Private app access token with required scopes

## Setup

### Step 1: Create HubSpot private app

1. Log in to HubSpot and navigate to **Settings > Integrations > Private Apps**
2. Click **Create a private app**
3. Name your app (e.g., "AgentC2 Integration")
4. Configure scopes:
    - `crm.objects.contacts.read`
    - `crm.objects.contacts.write`
    - `crm.objects.companies.read`
    - `crm.objects.companies.write`
    - `crm.objects.deals.read`
    - `crm.objects.deals.write`
    - `crm.objects.owners.read`
5. Save and copy the **Private app access token** (starts with `pat-na1-`)

### Step 2: Configure in AgentC2

**Via UI:**

1. Navigate to **Settings > Integrations**
2. Find **HubSpot** and click **Connect**
3. Enter your private app access token
4. Click **Test Connection** to verify credentials
5. Tools are automatically discovered and available

**Via API:**

```bash
curl -X POST https://agentc2.ai/agent/api/integrations/connections \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "providerKey": "hubspot",
    "name": "Production HubSpot",
    "credentials": {
      "HUBSPOT_ACCESS_TOKEN": "pat-na1-..."
    }
  }'
```

### Step 3: Attach tools to agents

List available HubSpot tools:

```bash
curl https://agentc2.ai/agent/api/tools \
  -H "Authorization: Bearer $TOKEN" \
  | jq '.tools[] | select(.id | startswith("hubspot_"))'
```

Attach tools to an agent:

```bash
curl -X POST https://agentc2.ai/agent/api/agents/sales-assistant/tools \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "toolId": "hubspot_hubspot-get-contact"
  }'
```

## Environment variables

| Variable               | Required | Description                                      |
| ---------------------- | -------- | ------------------------------------------------ |
| `HUBSPOT_ACCESS_TOKEN` | Yes      | Private app access token (format: `pat-na1-...`) |

<Callout type="warning">
    Store `HUBSPOT_ACCESS_TOKEN` securely. In production, use environment variables or encrypted
    credential storage. Never commit tokens to version control.
</Callout>

## Available tools

| Tool Name                          | Description                                   |
| ---------------------------------- | --------------------------------------------- |
| `hubspot_hubspot-get-contact`      | Retrieve contact details by ID                |
| `hubspot_hubspot-search-contacts`  | Search contacts by name, email, or properties |
| `hubspot_hubspot-create-contact`   | Create a new contact                          |
| `hubspot_hubspot-update-contact`   | Update contact properties                     |
| `hubspot_hubspot-get-company`      | Retrieve company details by ID                |
| `hubspot_hubspot-search-companies` | Search companies by name or properties        |
| `hubspot_hubspot-create-company`   | Create a new company                          |
| `hubspot_hubspot-update-company`   | Update company properties                     |
| `hubspot_hubspot-get-deal`         | Retrieve deal details by ID                   |
| `hubspot_hubspot-search-deals`     | Search deals by name, stage, or properties    |
| `hubspot_hubspot-create-deal`      | Create a new deal                             |
| `hubspot_hubspot-update-deal`      | Update deal properties and stage              |

Tool names follow the pattern `hubspot_{tool-name}`. Use the tools API to see the full list with parameter schemas.

## Common patterns

### Look up customer information

```typescript
// Agent receives: "What's the email for John Smith?"
// Agent calls:
{
  tool: "hubspot_hubspot-search-contacts",
  arguments: {
    query: "John Smith",
    limit: 1
  }
}
// Returns contact with email, phone, company, etc.
```

### Create deal from conversation

```typescript
// Agent receives: "Create a deal for Acme Corp worth $50k"
// Agent calls:
{
  tool: "hubspot_hubspot-get-company",
  arguments: { companyId: "acme-corp-id" }
}
// Then:
{
  tool: "hubspot_hubspot-create-deal",
  arguments: {
    name: "Acme Corp - Q1 2025",
    amount: 50000,
    companyId: "acme-corp-id",
    stage: "appointmentscheduled"
  }
}
```

### Update contact properties

```typescript
// Agent receives: "Mark john@example.com as a qualified lead"
// Agent calls:
{
  tool: "hubspot_hubspot-search-contacts",
  arguments: { query: "john@example.com" }
}
// Then:
{
  tool: "hubspot_hubspot-update-contact",
  arguments: {
    contactId: "contact-123",
    properties: {
      hs_lead_status: "QUALIFIED"
    }
  }
}
```

## Troubleshooting

| Problem                        | Solution                                                                                                |
| ------------------------------ | ------------------------------------------------------------------------------------------------------- |
| **"Authentication failed"**    | Verify `HUBSPOT_ACCESS_TOKEN` is correct and not expired. Regenerate token in HubSpot if needed.        |
| **"Tool not found"**           | Ensure connection is active (`isActive: true`). Test connection to refresh tool discovery.              |
| **"Insufficient permissions"** | Check private app scopes include required permissions (read/write for contacts, companies, deals).      |
| **"Rate limit exceeded"**      | HubSpot limits API calls per minute. Implement retry logic with exponential backoff.                    |
| **"Contact not found"**        | Verify contact ID or search query. Use `hubspot-search-contacts` to find IDs before calling get/update. |

## CRM-specific controls

### Deal stage management

HubSpot deals use stage-based pipelines. Common stages:

- `appointmentscheduled` - Initial contact
- `qualifiedtobuy` - Qualified lead
- `presentationscheduled` - Demo scheduled
- `decisionmakerboughtin` - Decision maker engaged
- `closedwon` - Deal won
- `closedlost` - Deal lost

Update deal stage:

```typescript
{
  tool: "hubspot_hubspot-update-deal",
  arguments: {
    dealId: "deal-123",
    properties: {
      dealstage: "closedwon"
    }
  }
}
```

### Custom properties

HubSpot supports custom properties. Access them via the `properties` object:

```typescript
{
  tool: "hubspot_hubspot-update-contact",
  arguments: {
    contactId: "contact-123",
    properties: {
      custom_field_name: "value"
    }
  }
}
```

Property names use HubSpot's internal format (e.g., `hs_lead_status`, `lifecyclestage`). Check HubSpot's property API or UI to find exact names.

### Owner assignment

Assign contacts, companies, or deals to HubSpot users:

```typescript
{
  tool: "hubspot_hubspot-update-deal",
  arguments: {
    dealId: "deal-123",
    properties: {
      hubspot_owner_id: "user-456"
    }
  }
}
```

Find owner IDs via `hubspot_hubspot-get-owners` (if available) or HubSpot's users API.

## Best practices

### 1. Use search before create

Always search for existing contacts/companies before creating duplicates:

```typescript
// Search first
const existing = await callTool("hubspot_hubspot-search-contacts", {
  query: "john@example.com"
});

if (existing.length > 0) {
  // Update existing
  await callTool("hubspot_hubspot-update-contact", {
    contactId: existing[0].id,
    properties: { ... }
  });
} else {
  // Create new
  await callTool("hubspot_hubspot-create-contact", {
    email: "john@example.com",
    properties: { ... }
  });
}
```

### 2. Batch operations

When updating multiple records, use search to get IDs, then update in parallel:

```typescript
const contacts = await callTool("hubspot_hubspot-search-contacts", {
    query: "qualified",
    limit: 10
});

await Promise.all(
    contacts.map((contact) =>
        callTool("hubspot_hubspot-update-contact", {
            contactId: contact.id,
            properties: { hs_lead_status: "QUALIFIED" }
        })
    )
);
```

### 3. Error handling

Handle HubSpot API errors gracefully:

```typescript
try {
    const result = await callTool("hubspot_hubspot-get-contact", {
        contactId: "invalid-id"
    });
} catch (error) {
    if (error.message.includes("not found")) {
        // Contact doesn't exist
        return "Contact not found. Would you like me to search for similar contacts?";
    } else if (error.message.includes("rate limit")) {
        // Rate limited
        return "HubSpot API rate limit reached. Please try again in a moment.";
    }
    throw error;
}
```
