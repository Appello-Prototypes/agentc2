# Google Calendar

Google Calendar integration enables agents to list events, create meetings, update schedules, and detect conflicts using Google OAuth2 authentication. The integration shares OAuth credentials with Gmail and Google Drive—one OAuth connection enables all three Google services.

## Overview

The Google Calendar integration provides OAuth-native tools that agents can call during conversations. Agents can search for events, create new meetings, update event details, delete events, and check for scheduling conflicts—all while respecting calendar permissions and availability.

<Callout type="info">
    Google Calendar integration uses the same Google OAuth2 connection as Gmail and Google Drive. If
    you've already connected Gmail, Calendar tools are automatically available.
</Callout>

## Prerequisites

- Google account with Google Calendar access
- Google Cloud project with OAuth2 credentials (same as Gmail)
- OAuth consent screen configured with Calendar scopes
- Required scopes authorized

## Setup

### Step 1: Configure Google Cloud OAuth

If you've already set up Gmail integration, skip to Step 2. Otherwise:

1. Go to [Google Cloud Console](https://console.cloud.google.com) and create or select a project
2. Navigate to **APIs & Services > Credentials**
3. Click **Create Credentials > OAuth client ID**
4. Application type: **Web application**
5. Authorized redirect URIs: `https://your-domain/api/integrations/gmail/callback`
6. Copy **Client ID** and **Client Secret**

### Step 2: Add Calendar scopes

1. Navigate to **APIs & Services > OAuth consent screen**
2. Add scopes:
    - `https://www.googleapis.com/auth/calendar.readonly` (read events)
    - `https://www.googleapis.com/auth/calendar.events` (read/write events)

<Callout type="tip">
    Use `calendar.readonly` if agents only need to read events. Use `calendar.events` for full CRUD
    access.
</Callout>

### Step 3: Configure in AgentC2

**Environment variables:**

```bash
GOOGLE_CLIENT_ID="your-client-id.apps.googleusercontent.com"
GOOGLE_CLIENT_SECRET="your-client-secret"
CREDENTIAL_ENCRYPTION_KEY="..." # Same as Gmail
```

**Via UI:**

1. Navigate to **Settings > Integrations**
2. Find **Google Calendar** (or **Gmail** - they share connection)
3. Click **Connect via Google Sign-In**
4. Authorize AgentC2 to access Google Calendar
5. Tokens are automatically stored encrypted
6. Tools are immediately available

**Note:** If Gmail is already connected, Calendar tools are automatically enabled. No separate connection needed.

### Step 4: Attach tools to agents

List available Google Calendar tools:

```bash
curl https://agentc2.ai/agent/api/tools \
  -H "Authorization: Bearer $TOKEN" \
  | jq '.tools[] | select(.id | startswith("google-calendar-"))'
```

Attach tools to an agent:

```bash
curl -X POST https://agentc2.ai/agent/api/agents/scheduling-assistant/tools \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "toolId": "google-calendar-list-events"
  }'
```

## Environment variables

| Variable                    | Required | Description                                           |
| --------------------------- | -------- | ----------------------------------------------------- |
| `GOOGLE_CLIENT_ID`          | Yes      | Google OAuth2 client ID (shared with Gmail/Drive)     |
| `GOOGLE_CLIENT_SECRET`      | Yes      | Google OAuth2 client secret (shared with Gmail/Drive) |
| `CREDENTIAL_ENCRYPTION_KEY` | Yes      | AES-256-GCM encryption key (same as Gmail)            |

## Available tools

| Tool Name                       | Description                           |
| ------------------------------- | ------------------------------------- |
| `google-calendar-list-events`   | List events in a date range           |
| `google-calendar-search-events` | Search events by query                |
| `google-calendar-get-event`     | Get event details by ID               |
| `google-calendar-create-event`  | Create a new calendar event           |
| `google-calendar-update-event`  | Update event details, time, attendees |
| `google-calendar-delete-event`  | Delete an event                       |

Tool names use the `google-calendar-` prefix. Use the tools API to see the full list with parameter schemas.

## Common patterns

### List upcoming events

```typescript
// Agent receives: "What meetings do I have today?"
// Agent calls:
{
  tool: "google-calendar-list-events",
  arguments: {
    timeMin: "2025-02-18T00:00:00Z",
    timeMax: "2025-02-18T23:59:59Z",
    maxResults: 10
  }
}
// Returns: Array of events with times, titles, attendees
```

### Create meeting

```typescript
// Agent receives: "Schedule a meeting with john@example.com tomorrow at 3pm"
// Agent calls:
{
  tool: "google-calendar-create-event",
  arguments: {
    summary: "Meeting with John",
    description: "Discuss project updates",
    start: {
      dateTime: "2025-02-19T15:00:00Z",
      timeZone: "America/New_York"
    },
    end: {
      dateTime: "2025-02-19T16:00:00Z",
      timeZone: "America/New_York"
    },
    attendees: [
      { email: "john@example.com" }
    ]
  }
}
// Returns: Created event with ID, webLink, etc.
```

### Check for conflicts

```typescript
// Agent receives: "Can I schedule a meeting tomorrow at 2pm?"
// Agent calls:
{
  tool: "google-calendar-list-events",
  arguments: {
    timeMin: "2025-02-19T14:00:00Z",
    timeMax: "2025-02-19T15:00:00Z"
  }
}
// If events exist, suggest alternative times
```

### Update event

```typescript
// Agent receives: "Move the 3pm meeting to 4pm"
// Agent calls:
{
  tool: "google-calendar-search-events",
  arguments: {
    query: "3pm meeting",
    timeMin: "2025-02-19T00:00:00Z",
    timeMax: "2025-02-19T23:59:59Z"
  }
}
// Then:
{
  tool: "google-calendar-update-event",
  arguments: {
    eventId: "event-id-from-search",
    start: {
      dateTime: "2025-02-19T16:00:00Z",
      timeZone: "America/New_York"
    },
    end: {
      dateTime: "2025-02-19T17:00:00Z",
      timeZone: "America/New_York"
    }
  }
}
```

## Event formats

### Date-time format

Events use RFC3339 format with timezone:

```typescript
{
  dateTime: "2025-02-19T15:00:00-05:00", // With offset
  timeZone: "America/New_York" // Or timezone name
}
```

### All-day events

Use `date` instead of `dateTime`:

```typescript
{
  start: { date: "2025-02-19" },
  end: { date: "2025-02-19" } // Same day for single-day event
}
```

### Recurring events

Use `recurrence` array:

```typescript
{
    recurrence: [
        "RRULE:FREQ=WEEKLY;BYDAY=MO,WE,FR", // Every Mon/Wed/Fri
        "RRULE:FREQ=DAILY;COUNT=5" // Daily for 5 occurrences
    ];
}
```

<Callout type="tip">
    RRULE syntax follows iCalendar RFC 5545. Use Google Calendar's UI to create recurring events and
    inspect the RRULE for reference.
</Callout>

## Conflict handling

### Check availability

Before creating events, check for conflicts:

```typescript
// Check for conflicts
const existing = await callTool("google-calendar-list-events", {
    timeMin: proposedStart,
    timeMax: proposedEnd
});

if (existing.length > 0) {
    // Suggest alternatives
    return `Time slot is busy. Available times: ${suggestAlternatives()}`;
}
```

### Free/busy queries

Use `freebusy` API (if available via tool) to check multiple calendars:

```typescript
// Check multiple attendees' availability
{
  tool: "google-calendar-check-availability", // If available
  arguments: {
    attendees: ["user1@example.com", "user2@example.com"],
    timeMin: "2025-02-19T09:00:00Z",
    timeMax: "2025-02-19T17:00:00Z"
  }
}
```

## Troubleshooting

| Problem                        | Solution                                                                                                                     |
| ------------------------------ | ---------------------------------------------------------------------------------------------------------------------------- |
| **"Event not found"**          | Verify event ID is correct. Check event exists and user has access (not deleted, in accessible calendar).                    |
| **"Invalid time format"**      | Use RFC3339 format with timezone. Ensure `dateTime` includes timezone offset or `timeZone` field.                            |
| **"Insufficient permissions"** | Ensure OAuth scopes include `calendar.events` (for write) or `calendar.readonly` (for read). Re-authorize if scopes changed. |
| **"Attendee not found"**       | Verify email addresses are correct. Google Calendar validates emails when sending invitations.                               |
| **"Rate limit exceeded"**      | Google Calendar API limits: 1,000 requests/100 seconds/user. Implement exponential backoff.                                  |

## Rate limiting

Google Calendar API quota limits:

- **Per-user rate limit**: 1,000 requests per 100 seconds per user
- **Daily quota**: Varies by project (default: 1 billion quota units)

Quota costs:

- `events.list`: 1 unit
- `events.get`: 1 unit
- `events.insert`: 50 units
- `events.update`: 50 units
- `events.delete`: 50 units

## Best practices

### 1. Use time ranges efficiently

Limit date ranges to reduce API calls:

```typescript
{
  tool: "google-calendar-list-events",
  arguments: {
    timeMin: "2025-02-19T00:00:00Z", // Start of day
    timeMax: "2025-02-19T23:59:59Z", // End of day
    maxResults: 50
  }
}
```

### 2. Handle timezones correctly

Always specify timezone to avoid confusion:

```typescript
{
  start: {
    dateTime: "2025-02-19T15:00:00",
    timeZone: "America/New_York" // Explicit timezone
  }
}
```

### 3. Validate before creating

Check for conflicts before creating events:

```typescript
// Check conflicts
const conflicts = await callTool("google-calendar-list-events", {
  timeMin: start,
  timeMax: end
});

if (conflicts.length === 0) {
  // Safe to create
  await callTool("google-calendar-create-event", { ... });
} else {
  // Suggest alternatives
  return `Time slot conflicts with: ${conflicts.map(e => e.summary).join(", ")}`;
}
```
