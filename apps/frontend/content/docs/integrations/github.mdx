# GitHub

GitHub integration connects your agents to GitHub repositories, enabling them to create issues, manage pull requests, search code, and track repository activity. The integration uses GitHub's hosted MCP server, providing standardized tool access to repository management features.

## Overview

The GitHub integration exposes repository operations as MCP tools that agents can call during conversations. Agents can create issues, open pull requests, search code, list repositories, and manage GitHub Actions workflows—all through natural language requests.

<Callout type="info">
    GitHub uses a hosted MCP server at `https://api.githubcopilot.com/mcp/`. No self-hosting
    required—just provide your GitHub Personal Access Token (PAT).
</Callout>

## Prerequisites

- GitHub account
- Personal Access Token (PAT) with required scopes
- Access to repositories you want to manage

## Setup

### Step 1: Create GitHub Personal Access Token

1. Log in to GitHub and navigate to **Settings > Developer settings > Personal access tokens > Tokens (classic)**
2. Click **Generate new token (classic)**
3. Name your token (e.g., "AgentC2 Integration")
4. Set expiration (recommended: 90 days or custom)
5. Select scopes:
    - `repo` - Full control of private repositories
    - `read:org` - Read org membership (if accessing org repos)
    - `workflow` - Update GitHub Action workflows
6. Click **Generate token**
7. Copy the token immediately (starts with `ghp_`)

<Callout type="warning">
    PATs are displayed only once. Save it securely—you'll need it for AgentC2 configuration. If
    lost, revoke and create a new token.
</Callout>

### Step 2: Configure in AgentC2

**Via UI:**

1. Navigate to **Settings > Integrations**
2. Find **GitHub** and click **Connect**
3. Enter your Personal Access Token
4. Click **Test Connection** to verify credentials
5. Tools are automatically discovered and available

**Via API:**

```bash
curl -X POST https://agentc2.ai/agent/api/integrations/connections \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "providerKey": "github",
    "name": "Production GitHub",
    "credentials": {
      "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_..."
    }
  }'
```

### Step 3: Attach tools to agents

List available GitHub tools:

```bash
curl https://agentc2.ai/agent/api/tools \
  -H "Authorization: Bearer $TOKEN" \
  | jq '.tools[] | select(.id | startswith("github_"))'
```

Attach tools to an agent:

```bash
curl -X POST https://agentc2.ai/agent/api/agents/dev-assistant/tools \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "toolId": "github_github-create-issue"
  }'
```

## Environment variables

| Variable                       | Required | Description                                      |
| ------------------------------ | -------- | ------------------------------------------------ |
| `GITHUB_PERSONAL_ACCESS_TOKEN` | Yes      | GitHub PAT with `repo` scope (format: `ghp_...`) |

<Callout type="tip">
    For organization repositories, ensure your PAT has `read:org` scope and you're a member of the
    organization with appropriate permissions.
</Callout>

## Available tools

| Tool Name                           | Description                                 |
| ----------------------------------- | ------------------------------------------- |
| `github_github-list-repos`          | List repositories accessible to the token   |
| `github_github-get-repo`            | Get repository details by owner/repo        |
| `github_github-create-issue`        | Create a new issue in a repository          |
| `github_github-get-issue`           | Get issue details by number                 |
| `github_github-list-issues`         | List issues in a repository                 |
| `github_github-update-issue`        | Update issue title, body, labels, assignees |
| `github_github-create-pull-request` | Open a pull request                         |
| `github_github-get-pull-request`    | Get PR details by number                    |
| `github_github-list-pull-requests`  | List PRs in a repository                    |
| `github_github-search-code`         | Search code across repositories             |
| `github_github-get-file-contents`   | Read file contents from repository          |
| `github_github-list-workflow-runs`  | List GitHub Actions workflow runs           |

Tool names follow the pattern `github_{tool-name}`. Use the tools API to see the full list with parameter schemas.

## Common patterns

### Create issue from conversation

```typescript
// Agent receives: "Create a bug issue in my-repo about login failing"
// Agent calls:
{
  tool: "github_github-create-issue",
  arguments: {
    owner: "username",
    repo: "my-repo",
    title: "Login failing",
    body: "Users report login failures after clicking the login button.",
    labels: ["bug"]
  }
}
// Returns: { number: 123, url: "https://github.com/...", ... }
```

### Search code

```typescript
// Agent receives: "Find where we handle authentication"
// Agent calls:
{
  tool: "github_github-search-code",
  arguments: {
    query: "authentication handler",
    repo: "username/my-repo"
  }
}
// Returns: Array of code matches with file paths and snippets
```

### List open pull requests

```typescript
// Agent receives: "Show me open PRs"
// Agent calls:
{
  tool: "github_github-list-pull-requests",
  arguments: {
    owner: "username",
    repo: "my-repo",
    state: "open"
  }
}
// Returns: Array of PRs with titles, authors, status
```

### Get file contents

```typescript
// Agent receives: "Show me the README"
// Agent calls:
{
  tool: "github_github-get-file-contents",
  arguments: {
    owner: "username",
    repo: "my-repo",
    path: "README.md",
    ref: "main" // Branch, tag, or commit SHA
  }
}
// Returns: File contents as string
```

## Repository references

GitHub tools use `owner/repo` format:

- **Owner**: GitHub username or organization name
- **Repo**: Repository name (without `.git`)

Examples:

- `owner: "octocat"`, `repo: "Hello-World"`
- `owner: "my-org"`, `repo: "project-name"`

<Callout type="info">
    For forks, use the original repository's owner/repo, not the fork's owner unless you want to
    work with the fork specifically.
</Callout>

## Code search

GitHub code search supports advanced queries:

```typescript
// Search in specific repository
{
  tool: "github_github-search-code",
  arguments: {
    query: "function authenticate",
    repo: "username/my-repo"
  }
}

// Search across all accessible repos
{
  tool: "github_github-search-code",
  arguments: {
    query: "language:typescript authentication"
  }
}
```

Search query syntax:

- `language:typescript` - Filter by language
- `repo:owner/repo` - Search in specific repo
- `path:src/` - Search in directory
- `extension:.ts` - Filter by file extension

<Callout type="tip">
    GitHub code search indexes may be slightly delayed. For real-time results, use
    `github-get-file-contents` to read specific files.
</Callout>

## Troubleshooting

| Problem                       | Solution                                                                                                   |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------- |
| **"Bad credentials"**         | Verify `GITHUB_PERSONAL_ACCESS_TOKEN` is correct and not expired. Regenerate token if needed.              |
| **"Not found"**               | Check repository owner/repo format. Ensure token has access to the repository.                             |
| **"Resource not accessible"** | Token lacks required scopes. Add `repo` scope for private repos, `read:org` for org repos.                 |
| **"Rate limit exceeded"**     | GitHub limits API calls (5,000/hour for authenticated requests). Implement retry with exponential backoff. |
| **"Invalid issue number"**    | Verify issue exists and is accessible. Check repository name and issue number format.                      |

## Rate limiting

GitHub API rate limits:

- **Authenticated requests**: 5,000 requests/hour
- **Search API**: 30 requests/minute

Handle rate limits gracefully:

```typescript
async function callWithRetry(tool, args, retries = 3) {
    try {
        return await callTool(tool, args);
    } catch (error) {
        if (error.status === 403 && error.message.includes("rate limit") && retries > 0) {
            const resetTime = error.headers["x-ratelimit-reset"];
            const waitMs = resetTime * 1000 - Date.now() + 1000; // Add 1s buffer
            await sleep(waitMs);
            return callWithRetry(tool, args, retries - 1);
        }
        throw error;
    }
}
```

## Best practices

### 1. Use repository caching

Cache repository lists to reduce API calls:

```typescript
// Cache repo list for 5 minutes
const repos =
    (await cache.get(`github:repos:${owner}`)) ||
    (await callTool("github_github-list-repos", { owner }));
await cache.set(`github:repos:${owner}`, repos, 300);
```

### 2. Validate repository access

Check repository exists and is accessible before operations:

```typescript
try {
    await callTool("github_github-get-repo", { owner, repo });
} catch (error) {
    if (error.status === 404) {
        return "Repository not found or not accessible.";
    }
    throw error;
}
```

### 3. Handle large responses

GitHub API paginates large results. Use pagination parameters:

```typescript
{
  tool: "github_github-list-issues",
  arguments: {
    owner: "username",
    repo: "my-repo",
    perPage: 30, // Max 100
    page: 1
  }
}
```
